Projet #1 (1ère année) — Réseau social pour entreprise (JumpX)

  • 📌 Titre du projet :
    Réseau social pour entreprise (JumpX)
  • 🏫 Contexte :
    Réalisé en 1ère année de formation au sein d’une équipe de trois. Nous n’avions que très peu de bases en développement au départ.
  • 🛠️ Technologies et outils utilisés :

    • Looping (MCD) : concevoir le Modèle Conceptuel de Données.
    • Gantt : planifier les tâches et organiser le travail.
    • XAMPP : serveur web local (Apache, MySQL, PHP...).
    • Eclipse : IDE, principalement pour Java mais adapté avec des plugins.
    • Laravel : framework PHP moderne pour structurer et accélérer le développement.
    • Composer : gestionnaire de dépendances (indispensable pour Laravel).
    • Node.js : installation de packages front‐end (via npm) si nécessaire.
    • Miro : conception de maquettes (wireframes) et brainstorming visuel en équipe.
  • 🧠 Objectif :
    Concevoir un réseau social interne favorisant la communication entre les collaborateurs (partage d’informations, commentaires, etc.).
  • 🎯 Mon rôle :
    Définir les besoins, clarifier les idées (macro‐idées) et coordonner les étapes de conception.
  • 🖼️ Images :
    • Logo de l’application (JumpX).
    • Maquettes “Écran de connexion” et “Écran de navigation”.

  • 🔗 Lien GitHub :
    Aucun dépôt public. Cette première année était surtout axée sur l’organisation, la conceptualisation et le démarrage des bases en développement.

Projets de 2ème Année

Projet #1 (2ème année) — RFTG (Gestion de location de DVD)

  • RFTG est une application de gestion de location de DVD développée pour répondre aux besoins d’un village de vacances. Réalisé en 2ème année dans le cadre de la formation, ce projet était mené en équipe.

    Il se décline en deux parties :
    • Une application web pour la gestion du stock (gérant, employés)
    • Une application mobile destinée aux clients (location de DVD)
  • 📌 Titre du projet :
    RFTG
  • 🏫 Contexte :
    Formation 2ème année, travail en équipe de [X] personnes, avec partage des tâches (backend, frontend, mobile…).
  • 🛠️ Technologies et outils utilisés :

    • XAMPP : serveur web local incluant Apache/MySQL/PHP
    • Laravel : framework PHP pour la partie web (frontend + MVC)
    • Composer : gestionnaire de dépendances PHP
    • Spring Boot : pour le backend Java (web services / API REST)
    • Android Studio : développement de l’application mobile (Java/Kotlin)
    • Visual Studio Code : IDE pour la partie Laravel et JavaScript
    • Miro : création des maquettes et diagrammes collaboratifs
    • Looping (MCD) : élaboration du Modèle Conceptuel de Données
  • 🧠 Objectif :
    Créer deux applications complémentaires :
    • Une application web pour gérer le stock des DVD (ajout, suppression, suivi des réservations)
    • Une application mobile pour permettre aux clients de louer des DVD facilement
  • 🎯 Mon rôle :
    J’ai contribué à la conception du web service (Spring Boot) avec l’équipe et j’ai personnellement développé le frontend sous Laravel, en utilisant Composer et Node pour installer et gérer les dépendances.
  • 🖼️ Images :
    Possibilités d’illustrations :
    • Capture d’écran de l’application web (interface de gestion du stock)
    • Vue de l’application mobile (location de DVD)
    • Schéma de l’architecture (API REST entre le web service et les clients)
    • MCD (Modèle Conceptuel de Données)
