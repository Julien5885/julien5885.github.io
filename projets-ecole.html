Projet #1 (1ère année) — Réseau social pour entreprise (JumpX)

  • 📌 Titre du projet :
    Réseau social pour entreprise (JumpX)
  • 🏫 Contexte :
    Réalisé en 1ère année de formation au sein d’une équipe de trois. Nous n’avions que très peu de bases en développement au départ.
  • 🛠️ Technologies et outils utilisés :

    • Looping (MCD) : concevoir le Modèle Conceptuel de Données.
    • Gantt : planifier les tâches et organiser le travail.
    • XAMPP : serveur web local (Apache, MySQL, PHP...).
    • Eclipse : IDE, principalement pour Java mais adapté avec des plugins.
    • Laravel : framework PHP moderne pour structurer et accélérer le développement.
    • Composer : gestionnaire de dépendances (indispensable pour Laravel).
    • Node.js : installation de packages front‐end (via npm) si nécessaire.
    • Miro : conception de maquettes (wireframes) et brainstorming visuel en équipe.
  • 🧠 Objectif :
    Concevoir un réseau social interne favorisant la communication entre les collaborateurs (partage d’informations, commentaires, etc.).
  • 🎯 Mon rôle :
    Définir les besoins, clarifier les idées (macro‐idées) et coordonner les étapes de conception.
  • 🖼️ Images :
    • Logo de l’application (JumpX).
    • Maquettes “Écran de connexion” et “Écran de navigation”.

  • 🔗 Lien GitHub :
    Aucun dépôt public. Cette première année était surtout axée sur l’organisation, la conceptualisation et le démarrage des bases en développement.

Projets de 2ème Année

Projet #1 (2ème année) — RFTG : Location de DVD

  • Titre :
    RFTG est un projet complet de gestion de location de DVD développé en deuxième année de BTS SIO, dans un contexte d’équipe étendue.
  • Structure du projet :
    Il se compose de deux applications reliées à un même backend Java (Spring Boot) et une base de données commune nommée Peach :
    • Mario : Application web dédiée au personnel (gestion du stock, inventaire)
    • Luigi : Application mobile à destination des clients (consultation et location de films)
  • Backend commun :
    Le backend développé avec Spring Boot expose des API REST accessibles depuis les deux interfaces. Le projet utilise également des logs et des outils de tests pour assurer la fiabilité des échanges.

Focus — Application web "Mario"

  • Contexte :
    Mario est l’application web utilisée par les gérants ou membres du staff d’un village de vacances pour gérer les stocks de DVD.
  • Fonctionnalités :
    • Connexion du personnel via la table staff (base Peach)
    • Consultation, modification et suppression de films
    • Ajout d’exemplaires (inventaire)
  • Contraintes techniques :
    • Abandon temporaire d’Eloquent (Laravel) pour éviter des incompatibilités avec la structure PHPMyAdmin
    • Connexion sans hashage de mots de passe (dans un but de test local)
    • Utilisation de GuzzleHTTP pour communiquer avec les Web Services Java
  • Suivi et méthodologie :
    • Maquettes créées avec Miro (connexion, liste des films, inventaire)
    • Modèle Conceptuel de Données conçu sur Looping
    • Tests manuels et logs Laravel pour assurer la traçabilité

    Développement complet du frontend Laravel (Mario), connexion à l’API Java (Spring Boot), intégration HTML/CSS des interfaces et test local via XAMPP.
