Projets de 1ère Année

Projet #1 (1ère année) — Comparaison de deux fichiers CSV avec Java (Eclipse)

  • 📌 Titre du projet :
    Comparaison de deux fichiers CSV
  • 🏫 Contexte :
    Réalisé lors de la 1ère année, avec un niveau débutant en Java et en programmation en général.
  • 🛠️ Technologies utilisées :

    • Eclipse : IDE utilisé pour coder en Java et tester le programme.
  • 🧠 Objectif :
    L’objectif était de nous familiariser avec les bases de la programmation en Java. Le projet consistait à comparer deux fichiers CSV pour repérer à la fois les lignes identiques et celles qui différaient.
  • 🎯 Mon rôle :
    J’ai commencé par analyser les données contenues dans chaque fichier, puis j’ai développé un programme Java capable de comparer ligne à ligne et de distinguer celles qui étaient communes de celles qui ne l’étaient pas.

    • Aperçu du code Java (interface Eclipse)

Projets de 2ème Année

Projet #1 (2ème année) — Recherche de vulnérabilités (Cellule Cyber)

  • 📌 Titre du projet :
    Recherche de vulnérabilité
  • 🏫 Contexte :
    Réalisé en 2ème année au sein d’une petite cellule Cyber (3 personnes). Ce projet nous à permis de découvrir la plateforme YesWeHack : Site web spécialisé dans la mise en relation d'entreprises et de chercheurs en sécurité (bug bounty). Un bounty correspond au site web d'une société qui souhaite vérifier la robustesse de leur application. Ce site nous à permis de nous familiariser à la détection de cinq types de vulnérabilités : Force Brute, Injection SQL, XSS réfléchi, IDOR et Open Redirect.
  • 🛠️ Technologies utilisées :

    • Kali Linux : distribution spécialisée pour les tests d’intrusion.
    • Burp Suite : outil d’audit de sécurité web (analyse, interception, automations).
    • OWASP ZAP (Zed Attack Proxy) : scan des URLs pour extraire tous les liens d’un site cible.
    • Python : extraction automatique du HTML de chaque page listée par ZAP.
    • DonjoYWH (ou Django/YesWeHack) : scripts/plug-ins pour gérer les scopes d’un bounty.
    • Eclipse : pour s’entraîner et développer des morceaux de code Java.
  • 🧠 Objectif :
    L’objectif pédagogique était de comprendre comment identifier rapidement les failles potentielles sur un site web (scope d’un bounty). En automatisant une partie du processus, nous pouvions repérer les formulaires sensibles et mener plus efficacement nos tests de sécurité.
  • 🎯 Mon rôle :
    J’ai développé un programme Java pour détecter et recenser différents types de formulaires (formulaires de commentaire, d’upload de fichiers, de login…). Les résultats de la recherche concernant l'analyse étaient stockés en base de données, puis exploités dans une application Laravel, afin de faciliter la vérification manuelle de la présence où non de formulaire sur les pages du site. Facilitant ainsi les tests de sécurité. (Les formulaires, où champs de dépots de fichiers étant les principales sources d'entrées d'attaques. Tests de sécurité réalisé via Burp Suite afin de vérifier la robustesse des sécurités en place.
  • 🖼️ Images :
    • Programme Python pour extraire le HTML de chaque URL
    • Programme Java de détection de formulaires
    • Intégration dans Laravel (avec Composer, XAMPP)
