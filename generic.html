Définition de la veille technologique

La veille technologique est un processus essentiel permettant d’anticiper les évolutions et innovations dans un domaine donné. Dans le cadre de mon BTS SIO option SLAM, elle me permet de suivre les avancées technologiques en développement web, en cybersécurité et en gestion des API, afin d’améliorer mes compétences et d’optimiser les solutions que je développe.

Mes projets, tels que JumpX (application mobile de réseau social), RFTG (gestion de location de DVD) et la recherche de vulnérabilités, nécessitent une veille continue sur les frameworks, les bonnes pratiques de sécurité et l’automatisation des analyses de code.

Méthodologie de veille : outils et méthodes

Sources de veille

  • Sites spécialisés : ZDNet, The Hacker News, Journal du Net, OWASP.
  • Conférences et webinaires : OWASP API Security, DEFCON, BlackHat.
  • Podcasts et vidéos : Les Experts Cyber, Computerphile, Hackersploit.

Sujets de veille pertinents

Développement Web & Frameworks

  • Laravel 11 : Nouveaux outils pour l’optimisation et la sécurité.
  • Spring Boot & API REST : Meilleures pratiques pour sécuriser les endpoints.
  • WebAssembly : Exécution d’applications lourdes dans le navigateur.

Cybersécurité et Recherche de vulnérabilités

  • Analyse de code statique : Automatisation de la recherche de failles dans les API.
  • Sécurité des JWT : Protéger les tokens et gérer les sessions de manière sécurisée.
  • Utilisation d’IA pour la cybersécurité : Détection automatique des anomalies dans le code.

Cas concrets liés à mes projets

Sécurité des API dans RFTG

Dans le cadre du projet RFTG, nous avons mis en place une API REST sécurisée avec Spring Boot. Une attention particulière a été portée sur l’authentification avec JWT, en appliquant les recommandations d’OWASP pour éviter les vulnérabilités IDOR.

Recherche de vulnérabilités dans le cadre professionnel

En entreprise, nous avons développé un programme d’analyse de sécurité en Java et Python, permettant d’automatiser la détection de failles dans les pages web. Cela inclut l’analyse des formulaires, des captchas et des configurations JWT, afin d’optimiser les audits de sécurité et de gagner un temps considérable sur les tests d’intrusion.

Optimisation du développement mobile avec JumpX

Le projet JumpX, notre première application mobile, nous a permis d’expérimenter le développement natif avec Java et de nous adapter aux évolutions des bonnes pratiques en architecture mobile.

Conclusion

Grâce à cette veille technologique, je suis en mesure d’anticiper les évolutions des technologies que j’utilise dans mes projets. Que ce soit pour la sécurisation des API, l’amélioration des processus DevOps ou le développement d’applications modernes, je peux adopter les meilleures pratiques et améliorer en continu mes compétences.
